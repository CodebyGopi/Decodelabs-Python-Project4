# System Vulnerability Checklist – README

## Overview

This project presents a **System Vulnerability Checklist** designed to identify, assess, and mitigate security weaknesses in computer systems. It covers common vulnerabilities in software, networks, user accounts, data protection, and physical security while providing practical recommendations to improve cybersecurity.

## Objectives

* Identify common security vulnerabilities.
* Evaluate overall system security.
* Analyze potential cyber risks.
* Recommend security controls and best practices.
* Increase cybersecurity awareness.

## Key Topics Covered

* Software Vulnerabilities
* Network Vulnerabilities
* Human Vulnerabilities
* Hardware Vulnerabilities
* Password Security
* Software Update Management
* User Account Security
* Antivirus & Firewall Protection
* Network Security
* Data Protection
* Safe User Practices
* Physical Security

## Risk Assessment

The project categorizes vulnerabilities into:

* **High Risk:** Weak passwords, outdated software, disabled firewalls.
* **Medium Risk:** Unsafe browsing, open ports, improper permissions.
* **Low Risk:** Minor configuration issues and infrequent password updates.

## Case Study

Includes analysis of the **WannaCry Ransomware Attack (2017)**, highlighting the importance of timely security updates, backups, and vulnerability management.

## Recommendations

* Use strong passwords and MFA.
* Keep systems and software updated.
* Enable antivirus and firewall protection.
* Perform regular backups.
* Conduct cybersecurity awareness training.
* Monitor system logs and network activity.

## Conclusion

Regular vulnerability assessments help organizations detect security gaps, reduce cyber risks, protect sensitive data, and strengthen their overall cybersecurity posture.
